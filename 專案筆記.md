這份文件記錄「怎麼操作這個專案」以及「本專案跟一般 Cloudflare Workers 專案不一樣的地方」。

## 一、這是什麼

出差代墊費用與委員交通方式管理網站。

- **架構**：Cloudflare Workers（後端 API）+ Static Assets（前端頁面）+ D1（資料庫）+ KV（收據圖片）
- **線上網址**：https://reimbursement-records.a815261s.workers.dev
- **登入方式**：單一密碼登入，登入後發一組有效 8 小時的 Cookie。

## 二、常見操作速查

| 想做的事 | 指令 |
|---------|------|
| 本機開發（啟動本地伺服器） | `npm run dev` |
| 整理程式排版（Prettier） | `npm run format` |
| 型別檢查 | `npm run typecheck` |
| 檢查＋建置測試（不真的部署） | `npm run check` |
| 套用資料表到「本機」資料庫 | `npm run db:migrate:local` |
| 套用資料表到「線上」資料庫 | `npm run db:migrate:remote` |
| **部署到線上（含型別檢查，推薦）** | `npm run deploy` |
| 部署到線上（不檢查，較快） | `npx wrangler deploy` |
| 看線上即時日誌 | `npx wrangler tail` |
| 改登入密碼（見下方說明） | `printf '%s' '新密碼' \| npx wrangler secret put ADMIN_PASSWORD` |

> 改完程式想更新線上版，只要跑 `npx wrangler deploy` 即可。

## 三、本專案的特殊之處 ⚠️

### 1. 收據圖片存在 KV，不是 R2
一般這類需求會用 R2 存圖片，但本專案為了**免費、且不必綁付款方式**，改用 **Cloudflare KV** 儲存收據圖片。

- 綁定名稱：`RECEIPTS`（KV Namespace）
- 程式裡的用法：上傳用 `env.RECEIPTS.put(key, arrayBuffer, { metadata: { contentType } })`；
  讀取用 `env.RECEIPTS.getWithMetadata(key, { type: "stream" })`，圖片的 MIME 類型存在 metadata 裡。
- **KV 免費額度**：儲存 1GB、每天寫入 1,000 次、讀取 10 萬次、單檔上限 25MB（程式另限制單張圖片 ≤ 8MB）。
- 若哪天圖片量爆掉，要改回 R2，需先在 Dashboard 啟用 R2（要綁卡），再把這幾處 API 換回 R2 寫法。

### 2. 密碼與金鑰用 Secret，不寫進程式
- 兩個機密：`ADMIN_PASSWORD`（登入密碼）、`SESSION_SECRET`（Cookie 簽章金鑰）。
- **線上**用 `wrangler secret put` 設定，存在 Cloudflare，不在程式碼裡。
- **本機**放在 `.dev.vars`（此檔已被 `.gitignore` 排除，**絕對不可上傳 GitHub**）。
- 範本檔 `.dev.vars.example` 只是格式參考，不含真實密碼，可以上傳。

**改線上登入密碼**：
```bash
printf '%s' '你的新密碼' | npx wrangler secret put ADMIN_PASSWORD
```

### 3. `worker-configuration.d.ts` 是自動產生的
- 這個型別檔由 `wrangler types` 依 `wrangler.jsonc` 產生，已被 `.gitignore` 排除。
- **只要你改了 `wrangler.jsonc`（新增/修改綁定），就要重跑** `npx wrangler types`，否則型別會對不上、TypeScript 報錯。

### 4. 程式碼曾是「超壓縮單行」風格
- `src/index.ts` 原本把很多句擠在同一行（機器沒問題、但人難讀）。
- 現已用 **Prettier** 格式化成正常多行。日後覺得又亂了，跑 `npm run format` 即可。

## 四、資料庫結構（重點）

- **expenses**：每筆代墊支出（日期、分類、項目、金額(以「分」為單位存整數)、付款方式、收據編號、備註、收據圖片欄位、關聯委員）。
- **members**：委員資料（姓名、交通方式、路線、交通費、備註）。

> 金額欄位 `amount_cents` 存的是「分」（整數），顯示時要除以 100，避免浮點誤差。

改資料表結構的流程：在 `migrations/` 新增一個 `.sql` 檔 → 先 `npm run db:migrate:local` 測本機 → 再對線上跑 `--remote` 版本。

## 五、第一次接手/換電腦要做的事

1. `npm install`
2. 複製 `.dev.vars.example` 成 `.dev.vars`，填入本機測試用的密碼與 session secret
3. `npm run db:migrate:local`（建本機資料表）
4. `npm run dev`
5. 確認 `wrangler whoami` 登入的是正確的 Cloudflare 帳號（`a815261s@gmail.com`）
